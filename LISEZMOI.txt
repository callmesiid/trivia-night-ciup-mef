═══════════════════════════════════════════
  POUR JOUER (sans installation)
═══════════════════════════════════════════

  1. Télécharge le repo (bouton "Download ZIP" sur GitHub)
  2. Décompresse le dossier où tu veux
  3. Ouvre n'importe quel fichier .html dans Chrome ou Edge
     → Tu peux jouer directement, aucune installation nécessaire

  ⚠ Utilise Chrome ou Edge (Firefox non compatible pour certaines fonctions)


═══════════════════════════════════════════
  POUR MODIFIER LES QUESTIONS
  (éditeur visuel — recommandé)
═══════════════════════════════════════════

  1. Ouvre customizer.html dans Chrome
  2. Clique sur "Connecter le dossier du projet"
  3. Sélectionne le dossier racine (celui qui contient data/)
  4. Tu peux ajouter, modifier, valider des questions directement
  5. Les sauvegardes s'écrivent automatiquement dans data/
  6. Lance ensuite build.py (voir ci-dessous) pour mettre les jeux à jour


═══════════════════════════════════════════
  POUR MODIFIER LES QUESTIONS
  (en ligne de commande — Python requis)
═══════════════════════════════════════════

  PRÉREQUIS : Python 3.8 ou supérieur

  ── Installation de Python ─────────────────────────────────────

    Windows :
      → https://www.python.org/downloads/
      → Coche "Add Python to PATH" pendant l'installation
      → Vérifie : ouvre un terminal et tape : python --version

    macOS :
      → https://www.python.org/downloads/
      → Ou via Homebrew : brew install python3

    Anaconda (si déjà installé) :
      → Ouvre "Anaconda Prompt"
      → python --version  (doit afficher 3.8+)

  ── Commandes disponibles ──────────────────────────────────────

    Depuis le dossier racine du projet :

    python build.py
      → Regénère tous les fichiers JS des jeux depuis data/
      → À lancer après avoir modifié des fichiers dans data/

    python build.py --custom
      → Même chose mais pour le mode "données personnalisées"

    python migrate.py
      → Migration one-shot (déjà effectuée, ne pas relancer)

    python migrate_assets.py
      → Migration des médias (déjà effectuée, ne pas relancer)


═══════════════════════════════════════════
  STRUCTURE DES FICHIERS
═══════════════════════════════════════════

  data/
  ├── risque_tout/   ← Questions du Risque Tout (JSON)
  ├── surenchere/    ← Thèmes et réponses de La Surenchère (JSON)
  ├── wagon/         ← Questions, bonus, malus du Wagon (JSON)
  ├── snapshot/      ← Photos du Snapshot (JSON)
  └── assets/
      ├── images/    ← Toutes les images de questions
      └── sons/      ← Tous les sons de questions

  assets/
  ├── Images/        ← Icônes et jetons partagés entre les jeux
  ├── images_front/  ← Visuels d'interface
  └── effets_sonores/← Sons de jeu partagés


═══════════════════════════════════════════
  PARTAGE DE QUESTIONS
═══════════════════════════════════════════

  Pour envoyer tes nouvelles questions à quelqu'un :
    1. Mets le dossier data/ sur Google Drive / WeTransfer
    2. L'autre personne le télécharge et remplace son dossier data/
    3. Elle lance : python build.py
    4. Les jeux sont à jour

  Pour recevoir des questions de quelqu'un d'autre : même procédure inverse.
